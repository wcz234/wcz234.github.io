前言

Git是现代软件开发必备的版本控制工具。但很多人只会基础的 add、commit、push，遇到复杂场景就懵了。

今天老王给大家系统讲讲Git的最佳实践！

fix 修复bug
docs 文档更新

refactor 重构
perf 性能优化
test 测试相关
chore 构建/工具相关

四、协作流程

Pull Request流程

  1. Fork或Clone仓库
  2. 创建功能分支
  3. 开发并提交
  4. 推送到远程
  5. 创建Pull Request
  6. Code Review
  7. 合并到主分支

Code Review要点

  • 代码逻辑是否正确
  • 是否有潜在bug
  • 代码风格是否一致
  • 是否有单元测试
  • 文档是否更新

五、实用技巧

1. 交互式rebase

可以：

  • 合并多个提交（squash）
  • 修改提交信息（reword）
  • 删除提交（drop）
  • 重排提交顺序

2. 查看文件历史

3. 搜索提交

总结

Git的强大之处在于它的灵活性，但这也意味着需要制定规范才能高效协作。希望这篇文章能帮助你建立良好的Git使用习惯！

记住：好的提交历史是项目的宝贵财富！
